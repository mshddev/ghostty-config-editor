Decisions already made

Eight key technical decisions

The load-bearing calls that shape every unit. These aren't asking for input — they're the spine the plan is built on.

KTD1 Consolidate before restyling

Land the token layer + shared components first, framework-neutral so the AppKit recorder shares them. Colors canonically dynamic NSColor so light/dark adapt.

KTD2 Serialize applyEdit as a gate

One in-flight write with same-key coalescing + FIFO, landed in Phase A. Gates every unit that raises apply frequency or runs tasks near the write path.

KTD3 Humanize values like names

Fallback chain: curated → true/false → On/Off → humanize. Numeric display gains scale/unit so 0–1 sliders read %. Display-only; stored values stay raw.

KTD4 One accent axis

Accent = current / selected / primary CTA only. Customized-state gets a non-accent tint; destructive reads system red explicitly.

KTD5 Hover strengthens, motion punctuates

One hover ButtonStyle (~4–6% fill, no movement) on visible controls only — never hover-only reveals; every hover ships a keyboard-focus equivalent. Motion reduce-motion gated.

KTD6 Theme previews from cached data

Mini-terminal renders only from already-parsed ThemeColors. Lists stay lazy; Dark/Light batch classify runs off-main, cancellable, with a scroll-perf gate.

KTD7 Verification is protocolized

Repackage before any visual check; click-driven manual verification; hover via pointer-move probes; six explicit end gates (light, keyboard, VO, resize, Dynamic Type, 463-theme perf).

KTD8 Pass-1 invariants are load-bearing

Picker seeding, commit-on-blur single write path, provider-identity guards, reset-all scoping — preserved by name in every touching unit. Exactly one Pass-1 decision revisited (the mid-row pill).
